### 4.2 성과 평가 체계

```python
# Fellow Performance Evaluation
class FellowEvaluation:
    """
    분기별 성과 평가 시스템
    """
    
    criteria = {
        "research_progress": 40,   # 연구 진행도
        "publication": 20,         # 논문/발표
        "participation": 20,       # 미팅 참여
        "collaboration": 10,       # 협업
        "initiative": 10,          # 자발성
    }
    
    def evaluate(self, fellow):
        score = 0
        
        # 연구 진행도 (40점)
        score += self.assess_research_progress(fellow)
        
        # 논문/발표 (20점)
        if fellow.has_publication():
            score += 20
        elif fellow.has_presentation():
            score += 10
        
        # 참여도 (20점)
        attendance_rate = fellow.meeting_attendance / fellow.total_meetings
        score += 20 * attendance_rate
        
        # 협업 (10점)
        score += min(fellow.collaboration_count * 2, 10)
        
        # 자발성 (10점)
        score += min(fellow.initiatives_count * 5, 10)
        
        return score
    
    def get_recommendation(self, score):
        if score >= 90:
            return "장학금 인상 권고"
        elif score >= 70:
            return "계속 지원"
        elif score >= 50:
            return "경고 및 개선 계획"
        else:
            return "프로그램 이탈 검토"
```

## 8. 리스크 관리

### 8.1 예상 리스크 및 대응

| 리스크 | 확률 | 영향 | 대응 전략 |
|--------|------|------|----------|
| **재원 부족** | 중 | 고 | 다각화된 재원 확보 |
| **학생 이탈** | 저 | 중 | 정기 면담, 동기 부여 |
| **멘토 참여 저조** | 저 | 고 | 인센티브 강화 |
| **연구 성과 부진** | 중 | 중 | 중간 점검, 연구 지원 |
| **장비 고장** | 저 | 중 | 예비 장비, 클라우드 백업 |

### 8.2 비상 계획

```
Contingency Planning
─────────────────────────────────

3. 장비 구매 연기

[학생 이탈 시]
1. 대기자 충원
2. 연구 과제 재배분
3. 잔여 예산 재분배

[멘토 참여 저조 시]
```
